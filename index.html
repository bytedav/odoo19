Docker Compose — Coolify

Configuración optimizada para producción con límites de memoria

▼
ℹ️ Usar en Coolify → New Service → Docker Compose. Las variables $SERVICE_USER_POSTGRES y $SERVICE_PASSWORD_POSTGRES las genera Coolify automáticamente.
⚠️ Ajustar shared_buffers y effective_cache_size según la RAM del VPS. Este compose está optimizado para servidor con 16GB compartido con otros servicios.
docker-compose.yml — Producción con Coolify
services: odoo: image: 'odoo:19.0' environment: - SERVICE_URL_ODOO_8069 - HOST=postgresql - USER=$SERVICE_USER_POSTGRES - PASSWORD=$SERVICE_PASSWORD_POSTGRES volumes: - 'odoo-web-data:/var/lib/odoo' - 'odoo-extra-addons:/mnt/extra-addons' - 'odoo-config:/etc/odoo' depends_on: postgresql: condition: service_healthy deploy: resources: limits: memory: 4G reservations: memory: 1G healthcheck: test: - CMD - curl - '-f' - 'http://127.0.0.1:8069' interval: 10s timeout: 10s retries: 10 restart: unless-stopped ulimits: nofile: soft: 65536 hard: 65536 postgresql: image: 'postgres:17' volumes: - 'postgresql-data:/var/lib/postgresql/data' environment: - POSTGRES_USER=$SERVICE_USER_POSTGRES - POSTGRES_PASSWORD=$SERVICE_PASSWORD_POSTGRES - POSTGRES_DB=postgres deploy: resources: limits: memory: 2G reservations: memory: 512M healthcheck: test: - CMD-SHELL - 'pg_isready -U $${POSTGRES_USER} -d postgres' interval: 5s timeout: 10s retries: 5 restart: unless-stopped command: > postgres -c max_connections=100 -c shared_buffers=512MB -c effective_cache_size=1536MB -c work_mem=16MB -c maintenance_work_mem=128MB -c checkpoint_completion_target=0.9 -c wal_buffers=16MB -c default_statistics_target=100 volumes: odoo-web-data: odoo-extra-addons: odoo-config: postgresql-data:
→ El volumen odoo-extra-addons persiste los módulos aunque se recree el contenedor. Los módulos instalados en /mnt/extra-addons sobreviven redeployments.
→ El volumen odoo-config persiste el odoo.conf. No tendrás que recrearlo en cada redeploy.

Verificar Contenedores

Identificar los nombres exactos antes de cualquier comando

▼
⚠️ Los nombres de contenedores cambian con cada redeploy. Siempre verifica primero con docker ps.
Terminal VPS
docker ps | grep -E "odoo|postgres"
→ Copia los nombres y pégalos en el formulario de arriba. El hash al final (ej: 4sgw4k) cambia en cada despliegue.
→ Para obtener las credenciales de PostgreSQL en Coolify:
Obtener credenciales
docker exec {DB} env | grep POSTGRES_USER && docker exec {DB} env | grep POSTGRES_PASSWORD
02

Instalar Git y Clonar Módulos

Clonar el repositorio de addons Enterprise en el contenedor

▼
⚠️ Siempre usar -u root. Sin esto dará "Permission denied" al instalar git.
Un solo comando — todo de una vez
docker exec -u root {WEB} bash -c "apt-get update -qq && apt-get install -y git -qq && git clone {REPO} /tmp/odoo19 && cp -r /tmp/odoo19/* /mnt/extra-addons/ && chown -R odoo:odoo /mnt/extra-addons && echo MODULOS_OK"
✅ Cuando veas MODULOS_OK al final el paso está completo. El repo tiene 43,000+ objetos — puede tardar 1-2 minutos.
03

Crear odoo.conf

Configurar el archivo de Odoo con addons_path y credenciales DB

▼
ℹ️ En Coolify el host de la DB es postgresql. En Dokploy es db. Verificar en el compose.
Crear odoo.conf
docker exec -u root {WEB} bash -c "mkdir -p /etc/odoo && cat > /etc/odoo/odoo.conf << 'EOF' [options] addons_path = /usr/lib/python3/dist-packages/odoo/addons,/mnt/extra-addons db_host = {HOST} db_port = 5432 db_user = {PG_USER} db_password = {PG_PASS} admin_passwd = {ADMIN} EOF"
04

Aplicar Parche AI (Bug stopPropagation)

Fix para el error Cannot read properties of undefined 'stopPropagation'

▼
🔧 Parche verificado — Mayo 2026
🐛 Bug en Odoo 19 Enterprise: el Composer del módulo AI llama a super.onFocusin() sin pasar el evento, causando crash al escribir en el panel de IA.
Archivo afectado: ai/static/src/discuss/composer_patch.js
docker exec -u root {WEB} bash -c "cat > /mnt/extra-addons/ai/static/src/discuss/composer_patch.js << 'EOF' import { Composer } from \"@mail/core/common/composer\"; import { patch } from \"@web/core/utils/patch\"; patch(Composer.prototype, { saveContent() { if (this.thread?.channel_type === \"ai_chat\") { return; } super.saveContent(); }, onFocusin(ev) { if (ev) { super.onFocusin(ev); } if (this.thread?.channel_type === \"ai_chat\" && ev?.target) { ev.target.select(); } }, get wysiwygConfig() { const config = super.wysiwygConfig; return { ...config, getRecordInfo: () => { return { resModel: this.thread?.model, resId: this.thread?.id, }; }, }; } }); EOF"

▼

07

Configurar BD — Expiración + Tours + Vector

Un solo comando que configura todo en la base de datos

▼
ℹ️ Ejecutar DESPUÉS de crear la base de datos en el paso anterior. Reemplaza {DB_NAME} con el nombre exacto de tu BD.
Configuración completa en un comando
docker exec {DB} psql -U {PG_USER} -d {DBNAME} -c " INSERT INTO ir_config_parameter (key, value) VALUES ('database.expiration_date', '2099-12-31'), ('database.expiration_reason', 'enterprise'), ('web.base.url.freeze', 'True'), ('website.show_odoo_branding', 'False') ON CONFLICT (key) DO UPDATE SET value = EXCLUDED.value; INSERT INTO res_users_web_tour_tour_rel (res_users_id, web_tour_tour_id) SELECT u.id, t.id FROM res_users u, web_tour_tour t ON CONFLICT DO NOTHING; UPDATE res_users SET odoobot_state = 'disabled' WHERE odoobot_state IS NOT NULL; CREATE EXTENSION IF NOT EXISTS vector; "
Reiniciar Odoo
docker restart {WEB}
✅ Resultado esperado: INSERT 0 4 en el primer bloque. Si gamification_challenge no existe es normal — ese módulo se instala después.

Instalar Módulos Enterprise

Orden correcto de instalación desde Aplicaciones

▼
ℹ️ Primero activa el modo desarrollador: agrega ?debug=1 a la URL. Luego ve a Aplicaciones → Actualizar lista de aplicaciones.

10

Quitar Marca de Agua Odoo

Eliminar "Con la tecnología de Odoo" del footer público

▼
ℹ️ Ejecutar después de instalar los módulos de sitio web. Primero identificar las vistas con el branding.
Buscar vistas con branding
docker exec {DB} psql -U {PG_USER} -d {DBNAME} -c "SELECT id, name FROM ir_ui_view WHERE arch_db::text LIKE '%El mejor%' OR arch_db::text LIKE '%sitio web gratuito%' OR arch_db::text LIKE '%Con la tecnología%';"
→ Desactivar SOLO las vistas de branding — NO las de mail, login o portal. Usar los IDs de la consulta anterior:
Desactivar vistas de branding (reemplaza los IDs)
docker exec {DB} psql -U {PG_USER} -d {DBNAME} -c "UPDATE ir_ui_view SET active = false WHERE id IN (ID1, ID2, ID3);"
Reiniciar
docker restart {WEB}
⚠️ Las vistas de Mail, Login Layout, Portal Sidebar contienen el texto pero NO deben desactivarse — afectarían el login y emails.

Referencia Rápida

Comandos frecuentes de mantenimiento

Ver logs en tiempo real

docker exec -u root {WEB} bash -c 'cd /mnt/extra-addons && git pull origin master' docker restart {WEB}

docker exec {DB} pg_dump -U {PG_USER} {DBNAME} > backup_$(date +%Y%m%d_%H%M).sql

docker exec -u odoo {WEB} odoo -d {DBNAME} --update=all --stop-after-init docker restart {WEB}

docker exec -u root {WEB} cat /etc/odoo/odoo.conf
