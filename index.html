01

Clonar módulos Enterprise

Instalar git y copiar addons al volumen extra-addons

⚠️Siempre usar -u root. Sin esto da "Permission denied" al instalar git.
clonar addons — un solo comando
docker exec -u root {WEB} bash -c "apt-get update -qq && apt-get install -y git -qq && git clone {REPO} /tmp/addons && cp -r /tmp/addons/* /mnt/extra-addons/ && chown -R odoo:odoo /mnt/extra-addons && echo ADDONS_OK"
✅Esperar ADDONS_OK al final. El repo tiene 43,000+ objetos — tarda 1-2 minutos.
02

odoo.conf + Parche AI + Dependencias

Configuración, fix del bug JS y librerías Python

crear odoo.conf
docker exec -u root {WEB} bash -c "mkdir -p /etc/odoo && cat > /etc/odoo/odoo.conf << 'EOF' [options] addons_path = /usr/lib/python3/dist-packages/odoo/addons,/mnt/extra-addons db_host = {HOST} db_port = 5432 db_user = {USER} db_password = {PASS} admin_passwd = {ADMIN} EOF"
🐛Bug Odoo 19: el Composer del módulo AI llama a super.onFocusin() sin pasar el evento — crash al escribir en el panel de IA. Este parche lo resuelve.
parche AI — fix stopPropagation (verificado mayo 2026)
docker exec -u root {WEB} bash -c "cat > /mnt/extra-addons/ai/static/src/discuss/composer_patch.js << 'EOF' import { Composer } from \"@mail/core/common/composer\"; import { patch } from \"@web/core/utils/patch\"; patch(Composer.prototype, { saveContent() { if (this.thread?.channel_type === \"ai_chat\") return; super.saveContent(); }, onFocusin(ev) { if (ev) super.onFocusin(ev); if (this.thread?.channel_type === \"ai_chat\" && ev?.target) ev.target.select(); }, get wysiwygConfig() { const config = super.wysiwygConfig; return { ...config, getRecordInfo: () => ({ resModel: this.thread?.model, resId: this.thread?.id }) }; } }); EOF"
instalar dependencias Python
docker exec -u root {WEB} pip3 install google-auth phonenumbers --break-system-packages

⚡ Configurar BD — EJECUTAR INMEDIATAMENTE

Expiración 2099 + tours + vector — antes de instalar módulos

🚨Ejecutar INMEDIATAMENTE después de crear la BD y ANTES de instalar cualquier módulo. Si no, aparece "Vencimiento: junio 2026".
configuración completa — un solo bloque
docker exec {DB} psql -U {USER} -d {DBNAME} -c " INSERT INTO ir_config_parameter (key, value) VALUES ('database.expiration_date', '2099-12-31'), ('database.expiration_reason', 'enterprise'), ('web.base.url.freeze', 'True'), ('website.show_odoo_branding', 'False') ON CONFLICT (key) DO UPDATE SET value = EXCLUDED.value; INSERT INTO res_users_web_tour_tour_rel (res_users_id, web_tour_tour_id) SELECT u.id, t.id FROM res_users u, web_tour_tour t ON CONFLICT DO NOTHING; UPDATE res_users SET odoobot_state = 'disabled' WHERE odoobot_state IS NOT NULL; CREATE EXTENSION IF NOT EXISTS vector; "
reiniciar
docker restart {WEB}

08

Quitar marca de agua Odoo

Eliminar "Con la tecnología de Odoo" del footer

buscar vistas con branding
docker exec {DB} psql -U {USER} -d {DBNAME} -c "SELECT id, name FROM ir_ui_view WHERE arch_db::text LIKE '%El mejor%' OR arch_db::text LIKE '%sitio web gratuito%' OR arch_db::text LIKE '%Con la tecnología%';"
Del resultado, desactivar SOLO las vistas de branding:

✅ Desactivar: Brand Promotion, Brand Promotion Message, website_sale.brand_promotion
❌ NO tocar: Mail notification layout, Login Layout, My Portal Record Sidebar, mail_notification_light
desactivar (reemplaza los IDs del resultado anterior)
docker exec {DB} psql -U {USER} -d {DBNAME} -c "UPDATE ir_ui_view SET active = false WHERE id IN (ID1, ID2, ID3);" docker restart {WEB}
MANTENIMIENTO
🔧

Comandos de referencia rápida

Logs, backups, actualizaciones

ver logs en tiempo real
docker logs -f {WEB}
backup de base de datos
docker exec {DB} pg_dump -U {USER} {DBNAME} > backup_$(date +%Y%m%d_%H%M).sql
actualizar módulos desde github
docker exec -u root {WEB} bash -c 'cd /mnt/extra-addons && git pull origin master' docker restart {WEB}
forzar actualización de todos los módulos
docker exec -u odoo {WEB} odoo -d {DBNAME} --update=all --stop-after-init docker restart {WEB}
verificar odoo.conf
docker exec -u root {WEB} cat /etc/odoo/odoo.conf
reconectar expiración (si vuelve a aparecer)
docker exec {DB} psql -U {USER} -d {DBNAME} -c "UPDATE ir_config_parameter SET value = '2099-12-31' WHERE key = 'database.expiration_date'; INSERT INTO ir_config_parameter (key,value) SELECT 'database.expiration_date','2099-12-31' WHERE NOT EXISTS (SELECT 1 FROM ir_config_parameter WHERE key='database.expiration_date');"
