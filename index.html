Asegúrate de mapear los volúmenes para que tus módulos Enterprise y configuraciones no se pierdan al reiniciar.
YAML
01

Inyectar Código Enterprise

Instalar git y clonar addons en el volumen

Usamos -u root porque el usuario por defecto de la imagen (odoo) no tiene permisos de escritura en carpetas de sistema.
BASH
docker exec -u root {WEB} bash -c "apt-get update -qq && apt-get install -y git -qq && git clone {REPO} /tmp/addons && cp -r /tmp/addons/* /mnt/extra-addons/ && chown -R odoo:odoo /mnt/extra-addons && echo 'ADDONS_READY'"
02

Configurar odoo.conf e IA

Enlace de módulos y Parche de IA

¡IMPORTANTE! El parche de IA resuelve un bug de la v19 que bloquea la interfaz al escribir. Debe aplicarse antes de iniciar sesión.
Generar odoo.conf
docker exec -u root {WEB} bash -c "mkdir -p /etc/odoo && cat > /etc/odoo/odoo.conf << 'EOF' [options] addons_path = /usr/lib/python3/dist-packages/odoo/addons,/mnt/extra-addons db_host = {HOST} db_port = 5432 db_user = {USER} db_password = {PASS} admin_passwd = admin_master_key EOF"
Aplicar Parche IA (Fix JS)
docker exec -u root {WEB} bash -c "cat > /mnt/extra-addons/ai/static/src/discuss/composer_patch.js << 'EOF' import { Composer } from \"@mail/core/common/composer\"; import { patch } from \"@web/core/utils/patch\"; patch(Composer.prototype, { saveContent() { if (this.thread?.channel_type === \"ai_chat\") return; super.saveContent(); }, onFocusin(ev) { if (ev) super.onFocusin(ev); if (this.thread?.channel_type === \"ai_chat\" && ev?.target) ev.target.select(); }, get wysiwygConfig() { const config = super.wysiwygConfig; return { ...config, getRecordInfo: () => ({ resModel: this.thread?.model, resId: this.thread?.id }) }; } }); EOF"
Instalar pgvector (Para memoria IA)
docker exec -u root {DB} bash -c "apt-get update -qq && apt-get install -y postgresql-17-pgvector -qq" && docker restart {WEB}

🚀 ACCIÓN REQUERIDA: Crear Base de Datos

Antes de continuar con los parches finales, debes crear la estructura en Postgres.

  1. Abre tu navegador en: http://tu-ip:8069
  2. Usa el Master Password definido (admin_master_key).
  3. Pon el nombre de base de datos: {DBNAME}.
  4. NO marques la casilla "Demo Data".
  5. Espera a que cargue el tablero principal.
Nota: Los siguientes comandos SQL darán error si la base de datos no ha sido creada primero en este paso.
FASE 2: TUNING Y PARCHES DE BASE DE DATOS
03

Activar Enterprise y Extensiones

Quitar expiración y habilitar Vector IA

SQL Tuning
docker exec {DB} psql -U {USER} -d {DBNAME} -c " INSERT INTO ir_config_parameter (key, value) VALUES ('database.expiration_date', '2099-12-31'), ('database.expiration_reason', 'enterprise'), ('web.base.url.freeze', 'True') ON CONFLICT (key) DO UPDATE SET value = EXCLUDED.value; UPDATE res_users SET odoobot_state = 'disabled'; CREATE EXTENSION IF NOT EXISTS vector;"
04

Dependencias PIP y Branding

Librerías necesarias y limpieza visual

Instalamos google-auth y phonenumbers para evitar errores en los módulos de Social y WhatsApp.
Instalar PIP Deps
docker exec -u root {WEB} bash -c "pip3 install google-auth phonenumbers --break-system-packages"
Quitar "Powered by Odoo"
docker exec {DB} psql -U {USER} -d {DBNAME} -c "UPDATE ir_ui_view SET active = false WHERE name LIKE '%brand_promotion%';"
